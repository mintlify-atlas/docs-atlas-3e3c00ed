XNNPACK supports three convolution operators, all using the NHWC tensor layout.

## 2D Convolution

`xnn_define_convolution_2d` defines a standard 2D convolution node, including grouped and depthwise variants.

```c
enum xnn_status xnn_define_convolution_2d(
  xnn_subgraph_t subgraph,
  uint32_t input_padding_top,
  uint32_t input_padding_right,
  uint32_t input_padding_bottom,
  uint32_t input_padding_left,
  uint32_t kernel_height,
  uint32_t kernel_width,
  uint32_t subsampling_height,
  uint32_t subsampling_width,
  uint32_t dilation_height,
  uint32_t dilation_width,
  uint32_t groups,
  size_t group_input_channels,
  size_t group_output_channels,
  float output_min,
  float output_max,
  uint32_t input_id,
  uint32_t filter_id,
  uint32_t bias_id,
  uint32_t output_id,
  uint32_t flags);
```

### Tensor shapes

| Tensor | Shape |
|---|---|
| Input | `[N, IH, IW, groups * group_input_channels]` |
| Filter | `[groups * group_output_channels, kernel_height, kernel_width, group_input_channels]` |
| Bias (optional) | `[groups * group_output_channels]` |
| Output | `[N, OH, OW, groups * group_output_channels]` |

### Parameters

<ParamField path="input_padding_top" type="uint32_t" required>
  Implicit zero-padding above the 2D input. Must be `0` when `XNN_FLAG_TENSORFLOW_SAME_PADDING` is set.
</ParamField>

<ParamField path="input_padding_right" type="uint32_t" required>
  Implicit zero-padding to the right of the 2D input. Must be `0` when `XNN_FLAG_TENSORFLOW_SAME_PADDING` is set.
</ParamField>

<ParamField path="input_padding_bottom" type="uint32_t" required>
  Implicit zero-padding below the 2D input. Must be `0` when `XNN_FLAG_TENSORFLOW_SAME_PADDING` is set.
</ParamField>

<ParamField path="input_padding_left" type="uint32_t" required>
  Implicit zero-padding to the left of the 2D input. Must be `0` when `XNN_FLAG_TENSORFLOW_SAME_PADDING` is set.
</ParamField>

<ParamField path="kernel_height" type="uint32_t" required>
  Height of the convolution kernel (filter).
</ParamField>

<ParamField path="kernel_width" type="uint32_t" required>
  Width of the convolution kernel (filter).
</ParamField>

<ParamField path="subsampling_height" type="uint32_t" required>
  Convolution stride in the height dimension.
</ParamField>

<ParamField path="subsampling_width" type="uint32_t" required>
  Convolution stride in the width dimension.
</ParamField>

<ParamField path="dilation_height" type="uint32_t" required>
  Dilation factor for kernel elements along the height dimension. Use `1` for standard (non-dilated) convolution.
</ParamField>

<ParamField path="dilation_width" type="uint32_t" required>
  Dilation factor for kernel elements along the width dimension.
</ParamField>

<ParamField path="groups" type="uint32_t" required>
  Number of convolution groups. Set to `1` for a standard convolution. Set equal to the number of input channels for a fully depthwise convolution.
</ParamField>

<ParamField path="group_input_channels" type="size_t" required>
  Number of input channels per group. Total input channels = `groups * group_input_channels`.
</ParamField>

<ParamField path="group_output_channels" type="size_t" required>
  Number of output channels per group. Total output channels = `groups * group_output_channels`.
</ParamField>

<ParamField path="output_min" type="float" required>
  Lower bound for clamping output values. Use `-INFINITY` to disable.
</ParamField>

<ParamField path="output_max" type="float" required>
  Upper bound for clamping output values. Use `+INFINITY` to disable.
</ParamField>

<ParamField path="input_id" type="uint32_t" required>
  Value ID of the input tensor in the subgraph.
</ParamField>

<ParamField path="filter_id" type="uint32_t" required>
  Value ID of the filter tensor in the subgraph.
</ParamField>

<ParamField path="bias_id" type="uint32_t" required>
  Value ID of the bias tensor, or `XNN_INVALID_VALUE_ID` to omit the bias.
</ParamField>

<ParamField path="output_id" type="uint32_t" required>
  Value ID of the output tensor in the subgraph.
</ParamField>

<ParamField path="flags" type="uint32_t" required>
  Bitmask of operator flags. See flags section below.
</ParamField>

### Flags

| Flag | Value | Description |
|---|---|---|
| `XNN_FLAG_TENSORFLOW_SAME_PADDING` | `0x00000004` | Use TensorFlow "SAME" padding. Padding is computed dynamically from the input size. All explicit padding parameters must be `0`. |

### Usage example

```c
// 3x3 convolution, 32 input channels, 64 output channels, stride 1
enum xnn_status status = xnn_define_convolution_2d(
  subgraph,
  /*input_padding_top=*/1,
  /*input_padding_right=*/1,
  /*input_padding_bottom=*/1,
  /*input_padding_left=*/1,
  /*kernel_height=*/3,
  /*kernel_width=*/3,
  /*subsampling_height=*/1,
  /*subsampling_width=*/1,
  /*dilation_height=*/1,
  /*dilation_width=*/1,
  /*groups=*/1,
  /*group_input_channels=*/32,
  /*group_output_channels=*/64,
  /*output_min=*/-INFINITY,
  /*output_max=*/+INFINITY,
  input_id,
  filter_id,
  bias_id,
  output_id,
  /*flags=*/0);
```

## Depthwise 2D Convolution

`xnn_define_depthwise_convolution_2d` defines a channel-separable depthwise convolution. Each input channel is convolved independently, and an optional depth multiplier expands the output channels.

```c
enum xnn_status xnn_define_depthwise_convolution_2d(
  xnn_subgraph_t subgraph,
  uint32_t input_padding_top,
  uint32_t input_padding_right,
  uint32_t input_padding_bottom,
  uint32_t input_padding_left,
  uint32_t kernel_height,
  uint32_t kernel_width,
  uint32_t subsampling_height,
  uint32_t subsampling_width,
  uint32_t dilation_height,
  uint32_t dilation_width,
  uint32_t depth_multiplier,
  size_t input_channels,
  float output_min,
  float output_max,
  uint32_t input_id,
  uint32_t filter_id,
  uint32_t bias_id,
  uint32_t output_id,
  uint32_t flags);
```

### Tensor shapes

| Tensor | Shape |
|---|---|
| Input | `[N, IH, IW, input_channels]` |
| Filter | `[1, kernel_height, kernel_width, input_channels * depth_multiplier]` |
| Bias (optional) | `[input_channels * depth_multiplier]` |
| Output | `[N, OH, OW, input_channels * depth_multiplier]` |

### Parameters

<ParamField path="depth_multiplier" type="uint32_t" required>
  Ratio of output channels to input channels. Set to `1` for a standard depthwise convolution (no channel expansion).
</ParamField>

<ParamField path="input_channels" type="size_t" required>
  Number of input channels.
</ParamField>

All padding, kernel, subsampling, and dilation parameters follow the same conventions as `xnn_define_convolution_2d`.

### Flags

| Flag | Value | Description |
|---|---|---|
| `XNN_FLAG_TENSORFLOW_SAME_PADDING` | `0x00000004` | Use TensorFlow "SAME" padding. |

### Usage example

```c
// Depthwise 3x3 convolution on 32 channels with depth_multiplier=1
enum xnn_status status = xnn_define_depthwise_convolution_2d(
  subgraph,
  /*input_padding_top=*/1,
  /*input_padding_right=*/1,
  /*input_padding_bottom=*/1,
  /*input_padding_left=*/1,
  /*kernel_height=*/3,
  /*kernel_width=*/3,
  /*subsampling_height=*/1,
  /*subsampling_width=*/1,
  /*dilation_height=*/1,
  /*dilation_width=*/1,
  /*depth_multiplier=*/1,
  /*input_channels=*/32,
  /*output_min=*/-INFINITY,
  /*output_max=*/+INFINITY,
  input_id,
  filter_id,
  bias_id,
  output_id,
  /*flags=*/0);
```

## 2D Deconvolution (Transposed Convolution)

`xnn_define_deconvolution_2d` defines a transposed convolution node, commonly used for upsampling in segmentation and generative networks.

```c
enum xnn_status xnn_define_deconvolution_2d(
  xnn_subgraph_t subgraph,
  uint32_t padding_top,
  uint32_t padding_right,
  uint32_t padding_bottom,
  uint32_t padding_left,
  uint32_t adjustment_height,
  uint32_t adjustment_width,
  uint32_t kernel_height,
  uint32_t kernel_width,
  uint32_t upsampling_height,
  uint32_t upsampling_width,
  uint32_t dilation_height,
  uint32_t dilation_width,
  uint32_t groups,
  size_t group_input_channels,
  size_t group_output_channels,
  float output_min,
  float output_max,
  uint32_t input_id,
  uint32_t filter_id,
  uint32_t bias_id,
  uint32_t output_id,
  uint32_t flags);
```

### Tensor shapes

| Tensor | Shape |
|---|---|
| Input | `[N, IH, IW, groups * group_input_channels]` |
| Filter | `[groups * group_output_channels, kernel_height, kernel_width, group_input_channels]` |
| Bias (optional) | `[groups * group_output_channels]` |
| Output | `[N, OH, OW, groups * group_output_channels]` |

### Parameters

<ParamField path="padding_top" type="uint32_t" required>
  Implicit padding above the 2D output data.
</ParamField>

<ParamField path="padding_right" type="uint32_t" required>
  Implicit padding to the right of the 2D output data.
</ParamField>

<ParamField path="padding_bottom" type="uint32_t" required>
  Implicit padding below the 2D output data.
</ParamField>

<ParamField path="padding_left" type="uint32_t" required>
  Implicit padding to the left of the 2D output data.
</ParamField>

<ParamField path="adjustment_height" type="uint32_t" required>
  Additional elements appended to the bottom of the 2D output data. Used to handle ambiguous output sizes when stride > 1.
</ParamField>

<ParamField path="adjustment_width" type="uint32_t" required>
  Additional elements appended to the right of the 2D output data.
</ParamField>

<ParamField path="upsampling_height" type="uint32_t" required>
  Upsampling factor in the height dimension (deconvolution height stride).
</ParamField>

<ParamField path="upsampling_width" type="uint32_t" required>
  Upsampling factor in the width dimension (deconvolution width stride).
</ParamField>

All other parameters (`kernel_height`, `kernel_width`, `dilation_height`, `dilation_width`, `groups`, `group_input_channels`, `group_output_channels`, `output_min`, `output_max`) follow the same conventions as `xnn_define_convolution_2d`.

### Usage example

```c
// 4x4 transposed convolution with stride 2 (2x upsampling)
enum xnn_status status = xnn_define_deconvolution_2d(
  subgraph,
  /*padding_top=*/1,
  /*padding_right=*/1,
  /*padding_bottom=*/1,
  /*padding_left=*/1,
  /*adjustment_height=*/0,
  /*adjustment_width=*/0,
  /*kernel_height=*/4,
  /*kernel_width=*/4,
  /*upsampling_height=*/2,
  /*upsampling_width=*/2,
  /*dilation_height=*/1,
  /*dilation_width=*/1,
  /*groups=*/1,
  /*group_input_channels=*/64,
  /*group_output_channels=*/32,
  /*output_min=*/-INFINITY,
  /*output_max=*/+INFINITY,
  input_id,
  filter_id,
  bias_id,
  output_id,
  /*flags=*/0);
```

## Common flags reference

| Flag | Value | Applies to |
|---|---|---|
| `XNN_FLAG_DEPTHWISE_CONVOLUTION` | `0x00000001` | Internal flag; marks a convolution as depthwise with HWGo filter layout |
| `XNN_FLAG_TENSORFLOW_SAME_PADDING` | `0x00000004` | Convolution, Depthwise Convolution |
| `XNN_FLAG_INPUT_NHWC` | `0x00000002` | Internal flag; indicates input is in NHWC layout when operator uses NCHW |

<Note>
`XNN_FLAG_DEPTHWISE_CONVOLUTION` and `XNN_FLAG_INPUT_NHWC` are internal flags used by the XNNPACK runtime. When using the Subgraph API, prefer `xnn_define_depthwise_convolution_2d` over passing this flag to `xnn_define_convolution_2d`.
</Note>
