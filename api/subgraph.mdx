---
title: "Subgraph"
description: "Create and manage XNNPACK Subgraph objects, which represent neural network models as a graph of Values (tensors) and Nodes (operators)."
---

A **Subgraph** is an abstract representation of a neural network model. It is the primary structure used to define the topology of a model before it is compiled into an executable Runtime. Subgraphs contain:

- **Values** — tensors identified by integer IDs, including external inputs/outputs and internal intermediates.
- **Nodes** — operator definitions (convolution, matmul, etc.) that connect Values.

Once a Subgraph is compiled into a Runtime, the Subgraph object can be deleted.

## Functions

### `xnn_create_subgraph`

```c
enum xnn_status xnn_create_subgraph(
  uint32_t external_value_ids,
  uint32_t flags,
  xnn_subgraph_t* subgraph_out);
```

Create an empty Subgraph object. Call this before defining any Values or Nodes.

<ParamField path="external_value_ids" type="uint32_t" required>
  The number of Value IDs to reserve for external tensors (inputs and outputs that will be provided by the caller at inference time). XNNPACK will not assign internal IDs in the range `[0, external_value_ids - 1]`, so you can use those IDs to refer to your model's inputs and outputs by fixed, predictable IDs.

  Pass `0` if you do not need to reserve any external Value IDs and will use `XNN_INVALID_VALUE_ID` when defining external tensors.
</ParamField>

<ParamField path="flags" type="uint32_t" required>
  Reserved for future use. Pass `0`.
</ParamField>

<ParamField path="subgraph_out" type="xnn_subgraph_t*" required>
  Pointer to a variable that receives the handle to the newly created Subgraph object on success. Must not be `NULL`.
</ParamField>

**Return values**

| Status | Meaning |
|--------|---------|
| `xnn_status_success` | Subgraph created successfully. `*subgraph_out` is valid. |
| `xnn_status_uninitialized` | `xnn_initialize` has not been called. |
| `xnn_status_out_of_memory` | Memory allocation for the Subgraph failed. |

---

### `xnn_delete_subgraph`

```c
enum xnn_status xnn_delete_subgraph(
  xnn_subgraph_t subgraph);
```

Destroy a Subgraph object and all associated Values and Nodes. After this call, the handle is invalid and must not be used.

<ParamField path="subgraph" type="xnn_subgraph_t" required>
  The Subgraph object to destroy. Must have been created by `xnn_create_subgraph`.
</ParamField>

**Return values**

| Status | Meaning |
|--------|---------|
| `xnn_status_success` | Subgraph and all associated resources destroyed. |

<Note>
  A Subgraph can be safely deleted after all Runtime objects compiled from it have been created. Runtime objects do not hold a reference to the Subgraph — they are fully independent once created.
</Note>

## Constants and flags

### `XNN_INVALID_VALUE_ID`

```c
#define XNN_INVALID_VALUE_ID UINT32_MAX
```

Sentinel value indicating an absent or unassigned Value ID. Pass this as `external_id` when defining a Value that does not require a specific external ID; XNNPACK will then allocate an internal ID and write it to `*id_out`.

---

### `XNN_MAX_TENSOR_DIMS`

```c
#define XNN_MAX_TENSOR_DIMS 6
```

The maximum number of dimensions a tensor can have. Tensors with more than 6 dimensions are not supported.

---

### `XNN_VALUE_FLAG_EXTERNAL_INPUT`

```c
#define XNN_VALUE_FLAG_EXTERNAL_INPUT 0x00000001
```

Mark a Value as an external input. Values with this flag must have their data pointer provided via `xnn_setup_runtime` (or `xnn_setup_runtime_v2`) before each inference.

---

### `XNN_VALUE_FLAG_EXTERNAL_OUTPUT`

```c
#define XNN_VALUE_FLAG_EXTERNAL_OUTPUT 0x00000002
```

Mark a Value as an external output. Values with this flag must have their data pointer provided via `xnn_setup_runtime` (or `xnn_setup_runtime_v2`) to receive inference results.

<Info>
  A Value can have both `XNN_VALUE_FLAG_EXTERNAL_INPUT` and `XNN_VALUE_FLAG_EXTERNAL_OUTPUT` set simultaneously, for in-place operations.
</Info>

---

## Reserving external Value IDs

When you create a Subgraph with `external_value_ids > 0`, XNNPACK reserves IDs `0` through `external_value_ids - 1` exclusively for your use. This lets you assign stable, predictable IDs to the model's inputs and outputs.

For example, if you reserve 3 external Value IDs and use:
- ID `0` → input tensor
- ID `1` → output tensor
- ID `2` → another external tensor

Then you pass `external_id = 0`, `1`, or `2` when calling `xnn_define_tensor_value` for each of those tensors. XNNPACK generates all other (internal) Value IDs starting from `external_value_ids`.

---

## Example

```c
#include <xnnpack.h>
#include <stdio.h>
#include <stdlib.h>

int main(void) {
  // 1. Initialize XNNPACK.
  if (xnn_initialize(NULL) != xnn_status_success) {
    fprintf(stderr, "Failed to initialize XNNPACK\n");
    return 1;
  }

  // 2. Create a Subgraph with 2 external Value IDs reserved:
  //    ID 0 = external input, ID 1 = external output.
  xnn_subgraph_t subgraph = NULL;
  enum xnn_status status = xnn_create_subgraph(
    /*external_value_ids=*/2,
    /*flags=*/0,
    &subgraph);
  if (status != xnn_status_success) {
    fprintf(stderr, "Failed to create subgraph: %d\n", status);
    return 1;
  }

  // 3. Define an external input tensor (FP32, shape [1, 224, 224, 3]).
  size_t input_dims[] = {1, 224, 224, 3};
  uint32_t input_id = XNN_INVALID_VALUE_ID;
  status = xnn_define_tensor_value(
    subgraph,
    xnn_datatype_fp32,
    /*num_dims=*/4, input_dims,
    /*data=*/NULL,
    /*external_id=*/0,
    XNN_VALUE_FLAG_EXTERNAL_INPUT,
    &input_id);
  if (status != xnn_status_success) {
    fprintf(stderr, "Failed to define input tensor: %d\n", status);
    xnn_delete_subgraph(subgraph);
    return 1;
  }

  // 4. Define an external output tensor (FP32, shape [1, 1000]).
  size_t output_dims[] = {1, 1000};
  uint32_t output_id = XNN_INVALID_VALUE_ID;
  status = xnn_define_tensor_value(
    subgraph,
    xnn_datatype_fp32,
    /*num_dims=*/2, output_dims,
    /*data=*/NULL,
    /*external_id=*/1,
    XNN_VALUE_FLAG_EXTERNAL_OUTPUT,
    &output_id);
  if (status != xnn_status_success) {
    fprintf(stderr, "Failed to define output tensor: %d\n", status);
    xnn_delete_subgraph(subgraph);
    return 1;
  }

  // 5. Define operator Nodes here (convolutions, activations, etc.) ...

  // 6. Compile into a Runtime, then delete the Subgraph.
  // xnn_runtime_t runtime = NULL;
  // xnn_create_runtime(subgraph, &runtime);
  xnn_delete_subgraph(subgraph);

  xnn_deinitialize();
  return 0;
}
```
