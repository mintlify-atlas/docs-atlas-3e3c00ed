## What is XNNPACK?

XNNPACK is a highly optimized solution for neural network inference on ARM, x86, WebAssembly, and RISC-V platforms. It provides low-level performance primitives designed to accelerate high-level machine learning frameworks rather than serving as a direct interface for deep learning practitioners.

The library focuses on delivering maximum throughput for feed-forward inference workloads. It achieves this through hand-written assembly micro-kernels, architecture-specific SIMD vectorization, and graph-level optimizations such as operator fusion, sparse inference, and memory layout reuse.

<Note>
XNNPACK is not intended for direct use by deep learning researchers. It is a backend acceleration library consumed by frameworks like TensorFlow Lite, PyTorch Mobile, and ONNX Runtime.
</Note>

## Supported architectures

XNNPACK targets a broad range of CPU and DSP architectures with platform-specific micro-kernels for each.

<Info>
All operators support **NHWC layout** and allow a custom stride along the channel dimension. This means operators can consume or produce a subset of channels in a tensor, providing zero-cost Channel Split and Channel Concatenation.
</Info>

## Performance

### Mobile phones (single-threaded)

Benchmarked on Android/ARM64 with Android NDK r21, March 2020.

| Model | Pixel, ms | Pixel 2, ms | Pixel 3a, ms |
|---|:---:|:---:|:---:|
| FP32 MobileNet v1 1.0X | 82 | 86 | 88 |
| FP32 MobileNet v2 1.0X | 49 | 53 | 55 |
| FP32 MobileNet v3 Large | 39 | 42 | 44 |
| FP32 MobileNet v3 Small | 12 | 14 | 14 |

| Model | Pixel, ms | Pixel 2, ms | Pixel 3a, ms |

## Ecosystem integrations

XNNPACK powers neural network inference across a wide range of ML frameworks and runtimes.

<CardGroup cols={2}>
  <Card title="TensorFlow Lite" href="https://blog.tensorflow.org/2020/07/accelerating-tensorflow-lite-xnnpack-integration.html">
    On-device inference for mobile and embedded devices. XNNPACK is the default acceleration backend for floating-point and quantized models.
  </Card>
  <Card title="TensorFlow.js" href="https://blog.tensorflow.org/2020/03/introducing-webassembly-backend-for-tensorflow-js.html">
    WebAssembly backend enabling accelerated in-browser inference using XNNPACK SIMD kernels.
  </Card>
  <Card title="PyTorch Mobile" href="https://pytorch.org/mobile">
    Mobile inference runtime for PyTorch models, using XNNPACK for CPU operator execution on ARM and x86 devices.
  </Card>
  <Card title="ONNX Runtime" href="https://onnxruntime.ai/docs/execution-providers/Xnnpack-ExecutionProvider.html">

  </Card>
  <Card title="ExecuTorch" href="https://pytorch.org/executorch-overview">
    PyTorch's on-device AI framework, with XNNPACK as a first-class backend for edge deployment.
  </Card>
  <Card title="MediaPipe" href="https://developers.googleblog.com/2020/01/mediapipe-on-web.html">
    Google's cross-platform ML pipeline framework, using XNNPACK for web and mobile compute.
  </Card>

Additional integrations include [Alibaba HALO](https://github.com/alibaba/heterogeneity-aware-lowering-and-optimization) (Heterogeneity-Aware Lowering and Optimization) and [Samsung ONE](https://github.com/Samsung/ONE) (On-device Neural Engine).

## Origins

XNNPACK is based on [QNNPACK](https://github.com/pytorch/QNNPACK), originally developed at Facebook. The codebase has diverged substantially over time and the XNNPACK API is no longer compatible with QNNPACK.
